# Index Bab B06

`B06` menutup fondasi `R01` dengan dua kemampuan penting: membuat program lebih tahan terhadap error dan mulai memecah kode ke modul yang rapi.

## Daftar Bab (C01-C12)

1. `C01-mengapa-error-terjadi-dan-cara-membacanya.md`  
   Tujuan: memahami bahwa error adalah sinyal perilaku program dan belajar membaca pesan error secara dasar.
2. `C02-referenceerror-typeerror-dan-syntaxerror-dasar.md`  
   Tujuan: mengenali tiga jenis error yang paling sering ditemui pemula.
3. `C03-try-catch-dasar.md`  
   Tujuan: memahami cara menangkap error runtime dengan `try...catch`.
4. `C04-finally-dan-alur-pembersihan-sederhana.md`  
   Tujuan: memahami kapan `finally` dipakai untuk langkah penutup yang harus tetap berjalan.
5. `C05-throw-dasar-untuk-validasi.md`  
   Tujuan: memahami cara melempar error secara sadar untuk validasi sederhana.
6. `C06-defensive-coding-dasar.md`  
   Tujuan: membangun kebiasaan mencegah error dengan pengecekan input dan asumsi yang lebih aman.
7. `C07-mengapa-modules-diperlukan.md`  
   Tujuan: memahami mengapa kode perlu dipecah ke file yang lebih kecil dan bertanggung jawab jelas.
8. `C08-export-dan-import-dasar.md`  
   Tujuan: memahami alur dasar berbagi kode antar file dengan `export` dan `import`.
9. `C09-named-export-dan-named-import.md`  
   Tujuan: memahami cara memakai named export untuk banyak nilai yang saling terkait.
10. `C10-default-export-dan-kapan-memakainya.md`  
    Tujuan: memahami default export, batasnya, dan kapan lebih tepat dipakai.
11. `C11-struktur-mini-project-modular-sederhana.md`  
    Tujuan: menyusun contoh mini project kecil dengan pemisahan file yang masuk akal untuk pemula.
12. `C12-checklist-debugging-error-dan-modules-dasar.md`  
    Tujuan: merangkum checklist praktis untuk membaca error dan melacak bug modul dasar.

## Urutan Baca

- Wajib berurutan dari `C01` sampai `C12`.
- `C01-C06` membangun fondasi error handling dari membaca error sampai defensive coding.
- `C07-C11` membangun fondasi modules dari alasan modularisasi sampai struktur mini project sederhana.
- `C12` berfungsi sebagai penutup praktis untuk merapikan pola debugging dasar.
