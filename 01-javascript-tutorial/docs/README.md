# Dokumentasi Buku

Folder `docs/` berisi dokumen pendukung yang mengatur **struktur dan aturan penulisan** untuk buku **JavaScript Tutorial**.

Dokumen dalam folder ini tidak berisi materi pembelajaran JavaScript.  
Materi utama tetap berada pada bagian isi buku.

Tujuan dari folder ini adalah menjaga agar seluruh materi dalam buku memiliki:

- struktur yang konsisten
- gaya penjelasan yang seragam
- ruang lingkup yang jelas
- kualitas penjelasan yang terjaga

Dengan adanya dokumen ini, buku dapat berkembang secara terstruktur tanpa kehilangan fokus.

---

# Fungsi Folder `docs/`

Folder ini digunakan untuk:

- mendefinisikan aturan penulisan materi
- menentukan struktur isi buku
- menyediakan template standar untuk setiap topik
- menjaga konsistensi antar bagian dalam buku

Dokumen dalam folder ini berfungsi sebagai **panduan editorial** ketika menulis atau memperbarui materi dalam buku.

---

# Isi Folder

Folder `docs/` berisi beberapa dokumen berikut.

## `writing-rules.md`

Menjelaskan aturan penulisan materi dalam buku ini.

Dokumen ini mencakup:

- fokus materi yang diperbolehkan
- batasan topik yang tidak dibahas di buku ini
- gaya penjelasan yang digunakan
- aturan penggunaan contoh kode

Tujuan dokumen ini adalah menjaga konsistensi gaya penulisan dalam seluruh isi buku.

---

## `content-structure.md`

Menjelaskan struktur isi buku.

Dokumen ini menentukan:

- daftar topik utama yang dibahas
- urutan pembahasan materi
- batas ruang lingkup setiap bagian

Dengan adanya dokumen ini, isi buku dapat berkembang secara terarah.

---

## `topic-template.md`

Berisi template standar untuk menulis setiap topik dalam buku.

Template ini membantu menjaga agar setiap materi memiliki struktur penjelasan yang konsisten, seperti:

- pengenalan konsep
- bentuk sintaks
- contoh kode
- penjelasan perilaku

Template ini digunakan sebagai acuan ketika menambahkan topik baru.

---

# Prinsip Umum

Dokumen dalam folder ini hanya berfungsi sebagai **panduan penulisan buku**.

Penjelasan konsep JavaScript seperti:

- variables
- functions
- objects
- arrays
- modules
- error handling

harus berada dalam **isi buku**, bukan dalam dokumen di folder `docs/`.

Dengan pemisahan ini, buku tetap terstruktur dengan jelas antara:

- **materi pembelajaran**
- **aturan penulisan**
