# AGENTS.md

Panduan ini menjelaskan cara memahami dan membantu repository ini agar pembaca manusia maupun AI tidak perlu menebak-nebak visi utamanya dari nol.

## Identitas Repository

Repository ini adalah **perpustakaan pembelajaran JavaScript**.

Ini bukan sekadar kumpulan snippet, catatan acak, atau dump dokumentasi. Repository ini dibuat untuk menyusun ulang domain JavaScript yang luas menjadi jalur belajar yang lebih manusiawi, bertahap, dan mudah dinavigasi.

Latar belakang utamanya sederhana: dokumentasi, referensi besar, dan bahkan specification-level material JavaScript sering terlalu padat bila dibaca langsung. Repository ini hadir sebagai bentuk penyusunan ulang yang lebih ramah belajar tanpa melepas presisi teknis.

## Analogi Struktur

Selalu pahami struktur repository ini dengan analogi berikut:

- root repository = perpustakaan
- folder level atas seperti `R01-*` = rak
- folder di dalam rak seperti `B01-*` = buku
- file materi di dalam `chapters/` = bab
- `examples/` = contoh runnable pendukung bab
- `assets/` = visual pendukung
- `docs/` = aturan, scope, dan panduan editorial pada level terkait

Analogi ini bukan hiasan. Ini adalah model organisasi utama repository.

## Tujuan Utama

Saat membantu repository ini, pegang tujuan berikut:

- mengubah domain JavaScript yang besar menjadi struktur belajar yang bertahap
- menjaga agar materi tetap presisi, tetapi lebih mudah dipahami daripada dokumentasi mentah
- membantu pembaca membangun mental model yang kuat, bukan sekadar hafal syntax atau API
- menjaga hubungan antar topik tetap jelas melalui rak, buku, dan urutan bab

## Prinsip Bantuan

Jika kamu adalah AI atau kolaborator yang membantu repository ini, gunakan prinsip berikut:

- Perlakukan repository ini sebagai kurikulum, bukan hanya kumpulan file markdown.
- Jaga konsistensi struktur perpustakaan: rak, buku, bab, examples, assets, dan docs.
- Utamakan kejelasan belajar, urutan materi, dan beban kognitif pembaca.
- Jangan menambahkan materi yang melompat terlalu jauh dari scope buku atau rak.
- Jangan mengubah nada repository menjadi terlalu dokumentatif, terlalu akademik, atau terlalu seperti catatan pribadi yang tidak terstruktur.
- Gunakan bahasa yang membantu pembaca memahami "apa", "bagaimana", dan "kenapa".
- Hormati aturan global di `docs/` dan aturan lokal di `docs/` level rak atau buku.

## Cara Mengambil Keputusan

Saat ragu, pilih arah yang paling mendukung pertanyaan berikut:

Apakah perubahan ini membuat perpustakaan JavaScript ini lebih mudah dipelajari, lebih konsisten, dan lebih nyaman dinavigasi oleh pembaca lain?

Jika jawabannya tidak jelas, evaluasi kembali sebelum menambah atau mengubah materi.

## Prioritas Saat Membantu

Urutan prioritas bantuan adalah:

1. menjaga visi repository sebagai perpustakaan pembelajaran JavaScript
2. menjaga struktur dan scope materi
3. menjaga kualitas pedagogi dan kejelasan penjelasan
4. menjaga ketepatan teknis
5. menjaga kerapian editorial, naming, dan konsistensi pendukung

## Catatan Penutup

Orang atau AI yang membaca repository ini seharusnya bisa langsung memahami bahwa ini adalah upaya menyusun "buku besar JavaScript" menjadi perpustakaan belajar yang lebih terarah.

Bila membantu repository ini, bantu sebagai penyusun perpustakaan pengetahuan, bukan hanya penulis file.
